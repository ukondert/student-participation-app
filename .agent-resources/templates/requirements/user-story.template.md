# User Story Template

## User Story ID
- Format: `<Workflowphase>-<Epic>-<laufende Nummer>`
- Beispiel: `Fahrt-buchen-Epic2-001`

## Titel
- Kurzbeschreibung der User Story

## Rolle
- Als <Rolle> ...

## Ziel/Wunsch
- ...möchte ich <Ziel/Wunsch> ...

## Nutzen
- ...um <Nutzen> zu erreichen.

---

## Epic
- Zugehöriges Epic: <Epic-Name>

## Workflowphase
- Zugeordnete Workflowphase aus der Story Map: <Workflowphase>

## Priorität
- must-have / should-have / could-have

## Story Points
- <Schätzung des Aufwands>

## Akzeptanzkriterien
